WP20 — Node Intelligence and Optical Parallelism
The Self-Aware Core Configuration Architecture
White Paper #20 · v1.0 · 2026-03-04

This white paper defines how Velsanet nodes recognize themselves and their neighbors, and how that self-awareness gives rise to intelligence.
It introduces the first formal node naming schema, distinguishes T4 rhizome structure from the proximity-based mesh of H6 and above, and explains how node intelligence — beginning at O8 (PAI) — enables hypercube-conformant connectivity and AI-requested optical parallelism.
Key topics:

Node name as a multi-dimensional structure
T4 rhizome vs. proximity mesh (H6 and above)
Self-awareness mechanism via MOCT Channel 1
H6: no Q layer · O8: Q6 or above · PAI / AAI / AsAI hierarchy
Hypercube-conformant connection prioritization
Parallel core support on AI request
Hop-by-hop E2E link formation without central computation

Link color semantics conform to WP13 — Node Color White Paper.
